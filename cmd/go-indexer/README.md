# Техническое задание: Система индексации файлов в Manticoresearch (Go-Indexer)

## 1. Общее описание
Создание консольного приложения на языке Go для сканирования файловой системы, извлечения метаданных и расчета контрольных сумм (SHA256) с последующим сохранением данных в поисковый движок Manticoresearch.

## 2. Системное окружение
*   **ОС:** Linux.
*   **Файловая система:** `/path/to/your/media`.
*   **База данных:** Manticoresearch (в docker).

## 3. Требования к данным (Schema)
Каждая запись в индексе `files` должна содержать следующие поля:


| Поле | Тип Manticore | Описание |
| :--- | :--- | :--- |
| `id` | `bigint` | Уникальный ID (хеш от полного пути файла) |
| `path` | `text` | Полный путь к файлу (индексируемый) |
| `name` | `text` | Имя файла (индексируемый) |
| `size` | `uint` | Размер файла в байтах |
| `mtime` | `uint` | Дата последнего изменения (Unix timestamp) |
| `ctime` | `uint` | Дата создания/изменения атрибутов (Unix timestamp) |
| `sha256` | `string` | Полный SHA256 хеш файла (hex) |
| `is_deleted` | `uint` | Флаг состояния (0 — активен, 1 — удален) |

## 4. Функциональные режимы работы
Скрипт должен поддерживать три раздельных или последовательных этапа:

### 4.1. Режим сканирования (Discovery)
*   Быстрый обход директорий без чтения содержимого файлов.
*   Расчет `id` на основе полного пути (алгоритм FNV-1a или MurmurHash3 64-bit).
*   Массовая вставка (`REPLACE`) метаданных (путь, имя, размер, mtime).
*   Обработка исключений на основе шаблонов (glob patterns).

### 4.2. Режим синхронизации (Sync/Cleanup)
*   **Удаление:** Пометка записей как `is_deleted=1` для файлов, которые более не существуют на диске.
*   **Обновление:** Если `mtime` или `size` файла на диске отличается от базы, поле `sha256` должно быть обнулено для последующего пересчета.

### 4.3. Режим хеширования (Hashing)
*   Выборка из БД записей с пустым полем `sha256`.
*   Последовательное или слабо-параллельное чтение файлов с HDD.
*   Расчет полной суммы SHA256.
*   Атомарное обновление записи в БД.

## 5. Конфигурация (YAML)
Программа должна считывать настройки из файла `config.yaml`:
```yaml
storage:
  scan_paths: 
    - "/path/to/your/media"
  exclude_patterns:
    - "**/node_modules/**"
    - "**/.git/**"
    - "*.tmp"

database:
  dsn: "127.0.0.1:9306" # Протокол MySQL для Manticore
  table_name: "files"
  batch_size: 1000

performance:
  hash_workers: 2 # Ограничение для HDD
  buffer_size: 1MB # Размер буфера при чтении файла для хеша
```

## 6. Нефункциональные требования
*   **HDD Friendly:** Ограничение количества потоков на чтение (не более 2 для зеркального RAID на HDD), чтобы избежать деградации производительности из-за перемещения головок.
*   **Надежность ID:** Использование 64-битного хеша для минимизации коллизий путей.
*   **Потоковая обработка:** Использование буферизованной записи в Manticore (batching) по 500-1000 записей.
*   **Память:** Скрипт не должен загружать весь список из 1.1 млн файлов в RAM; обработка должна идти итеративно.

## 7. Алгоритм работы (Pipeline)
1. Чтение конфига и подключение к Manticore.
2. Запуск `WalkDir` по целевым путям.
3. Для каждого файла:
    *   Проверка на `exclude_patterns`.
    *   Генерация `id`.
    *   Проверка метаданных в БД.
    *   `REPLACE` если данных нет или они изменились.
4. По завершении сканирования — запуск воркеров для расчета `sha256` для пустых строк.
